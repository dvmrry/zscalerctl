# zscalerctl v0.1.0

First security-reviewed, read-only release.

## Scope

- Read-only ZIA resource access for `locations` and `rule-labels`.
- OneAPI auth by default.
- Explicit ZIA legacy auth for read-only ZIA resources when OneAPI credentials
  are not available.
- Sanitized JSON output and restricted dump output with `manifest.json` and
  value-free `redaction_report.json`.
- Static shell completions for bash, zsh, and fish.

## Security Posture

- Raw SDK responses are not renderable by type.
- Resource output is allow-list projected before rendering or dump writing.
- Secret fields and unmodeled nested fields are dropped.
- Allowed strings pass through the redaction backstop.
- Free-text fields receive high-entropy token scanning for bare pasted secrets.
- Credential values remain wrapped until the SDK boundary.
- SDK environment, config-file, log, cache, and proxy discovery are bypassed and
  regression-checked.
- GitHub Actions are blocked from referencing live Zscaler credential variables.
- Vendored dependencies, `govulncheck`, staticcheck, gitleaks, Semgrep, docs
  scans, SDK-boundary checks, action pinning checks, and seed fuzz/property tests
  are part of the release gate.

## Known Limits

- Resource coverage is intentionally small.
- Table output is best-effort for quick human inspection; JSON and dump output
  are the stable automation surfaces.
- Live smoke remains a required release gate because fixtures cannot prove real
  tenant response shape or pagination behavior.
- Sanitized dumps remain confidential operational data.
